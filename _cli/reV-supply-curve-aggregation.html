reV-supply-curve-aggregation

reV Supply Curve Aggregation Command Line Interface

reV-supply-curve-aggregation [OPTIONS] COMMAND [ARGS]...

Options

--version

Show the version and exit.

-n, --name <name>

Job name. Default is “reV-agg”.

Default

reV-agg

-v, --verbose

Flag to turn on debug logging. Default is not verbose.

direct

reV Supply Curve Aggregation Summary CLI.

reV-supply-curve-aggregation direct [OPTIONS] COMMAND [ARGS]...

Options

-exf, --excl_fpath <excl_fpath>

Required Single exclusions file (.h5) or a list of exclusion files (.h5, .h5).

-gf, --gen_fpath <gen_fpath>

reV generation/econ output file. If not input, a generic exclusions aggregation will be run based on the res_fpath input.

-tm, --tm_dset <tm_dset>

Required Dataset in the exclusions file that maps the exclusions to the resource being analyzed.

-ef, --econ_fpath <econ_fpath>

reV econ output file (optional argument that can be included if reV gen and econ data are being used from different files.

-rf, --res_fpath <res_fpath>

Resource file (e.g. WTK or NSRDB), required if techmap dset is to be created or if gen_fpath is not input.

-exd, --excl_dict <excl_dict>

String representation of a dictionary of exclusion LayerMask arguments {layer: {kwarg: value}} where layer is a dataset in excl_fpath and kwarg can be “inclusion_range”, “exclude_values”, “include_values”, “inclusion_weights”, “force_inclusion_values”, “use_as_weights”, “exclude_nodata”, and/or “weight”.

-cd, --res_class_dset <res_class_dset>

Dataset to determine the resource class (must be in gen_fpath).

-cb, --res_class_bins <res_class_bins>

List of resource class bin edges.

-cf, --cf_dset <cf_dset>

Dataset containing capacity factor values to aggregate.

Default

cf_mean-means

-lc, --lcoe_dset <lcoe_dset>

Dataset containing lcoe values to aggregate.

Default

lcoe_fcr-means

-hd, --h5_dsets <h5_dsets>

Additional datasets from the source gen/econ h5 files to aggregate.

-d, --data_layers <data_layers>

String representation of a dictionary of additional data layers to include in the aggregation e.g. {“slope”: {“dset”: “srtm_slope”, “method”: “mean”}}

-r, --resolution <resolution>

Number of exclusion points along a squares edge to include in an aggregated supply curve point.

Default

64

-ea, --excl_area <excl_area>

Area of an exclusion pixel in km2. None will try to infer the area from the profile transform attribute in excl_fpath.

Default

0.0081

-pd, --power_density <power_density>

Power density in MW/km2 or filepath to variable power density csv file. None will attempt to infer a constant power density from the generation meta data technology. Variable power density csvs must have “gid” and “power_density” columns where gid is the resource gid (typically wtk or nsrdb gid). and the power_density column is in MW/km2.

-afk, --area_filter_kernel <area_filter_kernel>

Contiguous area filter kernel name (“queen”, “rook”).

Default

queen

-ma, --min_area <min_area>

Contiguous area filter minimum area, default is None (No minimum area filter).

-ff, --friction_fpath <friction_fpath>

Optional h5 filepath to friction surface data. Must match the exclusion shape/resolution and be paired with the –friction_dset input arg. The friction data creates a “mean_lcoe_friction” output which is the nominal LCOE multiplied by the friction data.

-fd, --friction_dset <friction_dset>

Optional friction surface dataset in friction_fpath.

-cs, --cap_cost_scale <cap_cost_scale>

Optional LCOE scaling equation to implement “economies of scale”. Equations must be in python string format and return a scalar value to multiply the capital cost by. Independent variables in the equation should match the names of the columns in the reV supply curve aggregation table. This equation may use numpy functions with the package prefix “np”. This will not affect offshore wind LCOE.

-o, --out_dir <out_dir>

Directory to save aggregation summary output.

Default

./

-mw, --max_workers <max_workers>

Number of cores to run summary on. None is all available cpus

-spw, --sites_per_worker <sites_per_worker>

Number of sc_points to summarize on each worker

Default

100

-ld, --log_dir <log_dir>

Directory to save aggregation logs.

Default

./logs/

-recalc, --recalc_lcoe <recalc_lcoe>

Flag to recalculate LCOE from the multi-year mean capacity factor and annual energy production data. The recalc (which happens by default because it represents a more “True” LCOE) requires several datasets to be aggregated in the h5_dsets input: system_capacity, fixed_charge_rate, capital_cost, fixed_operating_cost, and variable_operating_cost.

Default

True

-pre, --pre_extract_inclusions

Optional flag to pre-extract/compute the inclusion mask from the provided excl_dict, by default False. Typically faster to compute the inclusion mask on the fly with parallel workers.

-v, --verbose

Flag to turn on debug logging. Default is not verbose.

slurm

slurm (Eagle) submission tool for reV supply curve aggregation.

reV-supply-curve-aggregation direct slurm [OPTIONS]

Options

-a, --alloc <alloc>

Required SLURM allocation account name.

-wt, --walltime <walltime>

SLURM walltime request in hours. Default is 1.0

Default

1.0

-l, --feature <feature>

Additional flags for SLURM job. Format is “–qos=high” or “–depend=[state:job_id]”. Default is None.

-mem, --memory <memory>

SLURM node memory request in GB. Default is None

-mod, --module <module>

Module to load

-env, --conda_env <conda_env>

Conda env to activate

-sout, --stdout_path <stdout_path>

Subprocess standard output path. Default is in out_dir.

from-config

Run reV SC aggregation from a config file.

reV-supply-curve-aggregation from-config [OPTIONS]

Options

-c, --config_file <config_file>

Required reV aggregation configuration json file.

-v, --verbose

Flag to turn on debug logging. Default is not verbose.

valid-config-keys

Echo the valid SupplyCurveAggregation config keys

reV-supply-curve-aggregation valid-config-keys [OPTIONS]
